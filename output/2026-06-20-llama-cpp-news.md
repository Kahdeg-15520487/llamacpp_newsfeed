### [Server/API] — PR #24796: server: optimize get_token_probabilities

**Merged:** 2026-06-19 | **Author:** @angt

**Summary:** Replaces a full vocabulary sort with `std::partial_sort` in `get_token_probabilities`, so only the requested top-n tokens are ordered instead of the entire vocabulary. This directly benefits all server users who request `logprobs` in their completions — the server now spends far less time sorting logits before returning token probabilities.
