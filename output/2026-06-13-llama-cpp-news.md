### [Server/API] — PR #23871: ui: PWA support

**Summary:** Adds full Progressive Web App (PWA) support to the llama.cpp web UI, making it installable as a standalone app on desktop and mobile devices. ### [Performance] — PR #24384: mtmd: add batching API

**Merged:** 2026-06-12 | **Author:** @ngxson

**Summary:** Introduces a generic batching API to the multimodal transformer decoder (mtmd) system, enabling batched encoding of image/video chunks for significant speedups in llava-uhd-style vision models and video processing. The new API provides `mtmd_batch_add_chunk`, `mtmd_batch_encode`, and `mtmd_batch_get_output_embd` functions, where chunks are accumulated into a batch and encoded together using the 4th batch dimension. A `--mmproj-batch-size` CLI argument lets users control how many chunks are processed at once. The batching is wired up in `llama-server` and currently supports `build_vit()` based models. Users serving vision-language models through llama-server will see faster prompt processing when handling high-resolution images or multi-frame video inputs.

**Affected areas:** mtmd, server, multimodal, vision models

### [Backend] — PR #23770: vulkan: add pipeline barriers for memcpy read operations

**Merged:** 2026-06-12 | **Author:** @0cc4m

**Summary:** Inserts proper Vulkan pipeline barriers before and after memcpy operations on host-visible memory, fixing a correctness issue (#22930) where Intel Vulkan drivers could read stale data. The barriers ensure that GPU writes to buffer memory are fully visible to the host before a memcpy reads them, and that host writes are visible to the GPU after a memcpy writes to a buffer. This primarily affects Intel GPU users on the Vulkan backend who may have experienced silent data corruption or incorrect inference results. All Vulkan users benefit from the added synchronization correctness, though the impact is most noticeable on drivers with less aggressive implicit synchronization.

**Affected areas:** Vulkan, Intel GPU, memory synchronization
