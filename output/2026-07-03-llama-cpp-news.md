## 🔍 PR Details

### [Performance] — PR #23940: Remove redundant CUDA copies after gated_delta_net

**Merged:** 2026-07-03 | **Author:** @gaugarg-nv

**Summary:** Eliminates redundant CUDA memory copy operations in the Gated Delta Network (GDN) path by making the CUDA GDN kernel write recurrent state snapshots directly into the target cache buffer, bypassing the intermediate tail output and subsequent `ggml_cuda_cpy` calls. Previously, for MTP draft length 3, the target decode phase (K=4) incurred 4 extra copy kernels per iteration. Benchmarks on a DGX Spark with Qwen3.6-35B-A3B-UD-Q4_K_M show a **3% decode phase speedup without MTP** and **~4% average throughput gain with MTP** (from ~78 t/s to ~83 t/s worst-case, ~101 t/s to ~105 t/s best-case). Users running speculative decoding with MTP on MoE/GDN models (Qwen3.6, Step3.5, etc.) will benefit immediately — no new flags or configuration changes required.

**Affected areas:** CUDA backend, Gated Delta Network (GDN), speculative decoding (MTP), `ggml_cuda_cpy` fusion

### [Build/CI] — PR #25218: vendor : update cpp-httplib to 0.49.0

**Merged:** 2026-07-03 | **Author:** @cabelo

**Summary:** Updates the vendored `cpp-httplib` library from version 0.48.0 to 0.49.0, bringing important security fixes including proper escaping of `"`, CR, and LF characters in multipart/form-data field names and filenames to prevent header injection attacks.
