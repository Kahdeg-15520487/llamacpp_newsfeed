## 🔍 PR Details

### [Model Support] — PR #25202: llama: fix quantized kv-cache for dsv4

**Merged:** 2026-07-07 | **Author:** @am17an

**Summary:** DeepSeek-V4's compressed CSA and HCA attention caches did not properly apply the Hadamard incoherence rotation when quantized KV cache (`--cache-type-k q8_0`) is used. This PR adds rotation support to the CSA and HCA sparse attention paths in the DeepSeek-V4 architecture, making quantized KV cache functionally correct with the model's compressed state management. A side effect is that concatenated caches must now be aligned to 512-element boundaries, requiring some additional padding. The `ggml_mul_mat_aux` helper was refactored into a shared `llama_mul_mat_hadamard` utility in `llama-impl.h` to enable reuse across both the sparse and raw attention paths. DeepSeek-V4 users who enable quantized KV cache to reduce VRAM usage will now get correct generation instead of silently corrupted outputs.

**Affected areas:** deepseek4 model, llama-kv-cache, llama-graph, Hadamard rotation, CSA/HCA/LID compressed attention

### [Backend] — PR #25231: [SYCL] fix unsupported UT cases of CONT & CPY

**Merged:** 2026-07-07 | **Author:** @arthw

**Summary:** Fixes previously unsupported unit test cases for the CONT (contiguous) and CPY (copy) operations in the SYCL backend. The changes enable support for bf16 data type in CONT operations across various tensor shapes, and significantly expand CPY coverage to include bf16, q1_0, mxpfp4, nvfp4, q2_K through q6_K, iq4_nl, iq4_xs, and other quantized types with permuted src tensors. Conversion paths from f32→bf16, f16→bf16, and bf16→f32/f16 were also enabled. New large-tensor MUL_MAT test cases (2880×2880) for q4_0, q8_0, and mxfp4 types were added. SYCL backend test coverage is substantially improved, and the ops.md/SYCL.csv documentation is updated to reflect the newly passing tests. Intel Arc GPU users and other SYCL backend users benefit from broader operator support.

**Affected areas:** SYCL backend, CONT operation, CPY operation, ops.md, SYCL.csv documentation
