### [Server/API] — PR #25225: UI — Improve performance when streaming

**Merged:** 2026-07-03 | **Author:** @ntowle

**Summary:** Optimizes the chat web UI's streaming rendering pipeline by memoizing `nodeMap` and sibling-map computations. Previously, on every stream event, the UI rebuilt the full message node-map and sibling map for every `filteredMessage` — resulting in O(n²) CPU burn proportional to conversation length. After this fix, the node-map and sibling maps are rebuilt only on structural changes (edit, regenerate, delete), while streaming events read from the cached maps. Benchmarks show total scripting time dropping from 9235 ms to 3056 ms — a 3× reduction. Users with long conversations in the llama-server web UI will see noticeably snappier streaming and lower browser CPU usage. No new flags or configuration changes required.

**Affected areas:** server UI, chat interface, streaming performance, JavaScript memoization

### [Server/API] — PR #25238: Chat — Trim messages sent to StepFun parser (fixes long reasoning loops)

**Merged:** 2026-07-03 | **Author:** @pwilkin

**Summary:** Fixes an infinite reasoning loop issue with StepFun models by trimming whitespace from `content` and `reasoning_content` fields before passing them to the StepFun-specific chat template parser. Previously, auto-trimming had been removed from the template parser because it interfered with other models' behavior — but StepFun models specifically break without it, entering long reasoning loops. This adds a targeted workaround that trims these two fields only for StepFun models. Users running StepFun models (e.g., Step3.5, Step3.7) via llama-server or llama-cli will see correct behavior with no infinite reasoning loops. Fixes issue #24181. No new flags or configuration changes required.

**Affected areas:** chat template parser, StepFun model support, reasoning content trimming
