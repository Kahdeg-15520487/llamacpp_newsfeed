### [Performance/Backend] — PR #20793: Sched: Reintroduce less synchronizations between token, with fixed pipeline parallelism

**Merged:** 2026-06-26 | **Author:** @aendk

**Summary:** Reintroduces a CUDA performance optimization that was previously rolled back due to multi-GPU pipeline parallelism bugs. The change moves scheduling from the `sassassasg` pattern (sync, async copy, sync, ...) to the more efficient `saaasg` pattern (sync, async copy, async copy, async copy, sync, graph execution), eliminating two superfluous per-token synchronizations between memory copies and graph execution. The fix enrolls pipeline parallelism into the same single synchronization between async copies and graph execution that the single-GPU path already uses, ensuring correct perplexity scores on multi-GPU setups. Windows users see the biggest benefit, and all CUDA users with single or multi-GPU configurations should see improved token throughput.

**Affected areas:** CUDA backend, GPU scheduling, multi-GPU pipeline parallelism, per-token synchronization
