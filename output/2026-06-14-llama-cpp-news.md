### [Performance] — PR #18039: feat: add EAGLE3 speculative decoding support

**Merged:** 2026-06-12 | **Author:** @ruixiang63

**Summary:** Adds comprehensive support for EAGLE3 speculative decoding, the current state-of-the-art method for accelerating LLM inference. EAGLE3 uses an encoder-decoder architecture that extracts features from the target model at specific layers, compresses them through a feature fusion layer, and generates draft tokens with a single-layer decoder — achieving 2–3× speedups on compatible models. This is a collaboration between NVIDIA and GGML teams; it integrates the `LLM_ARCH_EAGLE3` architecture, adds `--spec-type draft-eagle3` and `--spec-draft-n-max` flags to `llama-server`, provides the `convert_hf_to_gguf.py` conversion script for EAGLE3 draft models, and supports a wide range of models including LLaMA 3.1/3.3, Qwen3 (various sizes), Gemma 4, and GPT-OSS variants. Users running `llama-server` with `-md <draft_model.gguf> --spec-type draft-eagle3 --spec-draft-n-max 8` can expect significant throughput improvements, especially on code generation and reasoning tasks where acceptance rates are highest.

**Affected areas:** speculative decoding, llama-server, GGML backend, model conversion

### [Model Support] — PR #24260: Add arch support for cohere2-MoE

**Merged:** 2026-06-13 | **Author:** @michaelw9999

**Summary:** Adds a new `cohere2_moe` architecture supporting CohereLab's `North-Mini-Code-1.0` MoE coding model (30B-A3B), an early preview of a fast coding-specialized model with an optional multi-token prediction (MTP) head. The implementation includes `cohere2_moe` arch wiring in the model loader, GGUF conversion via `convert_hf_to_gguf.py`, and support for NVFP4 quantization on both weights and input scales (with a fix backported to the existing `cohere2` arch). The model has been tested to pass `test-llama-arches` on both CUDA (RTX 5090) and CPU, achieving ~11,120 t/s on prompt processing (pp512) in NVFP4, and working tool calling through llama-server. Users can now run this experimental 30B-A3B coding model locally using `-hf` with the converted GGUFs available on Hugging Face.

**Affected areas:** model architecture, MoE, quantization, conversion scripts
