### [Backend/Bug Fix] — PR #24909: Vulkan: Apply bias before softmax in FA, to avoid overflow

**Merged:** 2026-06-24 | **Author:** @jeffbolznv

**Summary:** Fixes an fp16 overflow issue in the Vulkan backend's flash attention (FA) scalar and CM1 paths by applying a bias offset before the softmax computation. The bias was already present in the CM2 (cooperative matrix) path but was missing from the scalar and CM1 paths. This fixes incorrect image generation in stable-diffusion.cpp (referenced issue #1678). Users running image generation or any models using flash attention on the Vulkan backend will see corrected outputs where previously FP16 overflow could cause artifacts or bad results.

**Affected areas:** Vulkan backend, flash attention, FP16 overflow, image generation
