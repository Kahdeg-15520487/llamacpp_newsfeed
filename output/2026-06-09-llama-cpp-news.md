### [Backend] — PR #24206: GGML_OP_COL2IM_1D CPU (voice vocoder upsampling)

**Merged:** 2026-06-09 | **Author:** @ServeurpersoCom

**Summary:** Adds a new GGML operation `GGML_OP_COL2IM_1D` for the CPU backend, implementing the overlap-add step of ConvTranspose1d as a GEMM + col2im decomposition. This splits the existing naive conv_transpose_1d kernel into a matmul (which can use quantized weights) followed by a thin, memory-bound scatter, enabling F16/BF16/quantized weight support for neural audio vocoders. This is the upsampling primitive used in acestep.cpp (music generation), omnivoice.cpp (multilingual TTS), and qwentts.cpp (Qwen3-TTS). The CUDA follow-up will be validated against the same test grid.

**Affected areas:** ggml-cpu, ops

### [Server/API] — PR #24190: Always export idle slots to RAM

**Merged:** 2026-06-09 | **Author:** @fiesh

**Summary:** Fixes a bug where `--cache-idle-slots` without unified KV cache would not write a slot's VRAM cache to RAM when the slot became idle. This caused needless reprocessing in another slot when a previously cached prompt was re-requested. The fix ensures that idle slots are always exported to RAM (without evicting from VRAM), eliminating superfluous preprocessing with parallel slots. Closes #22942.

**Affected areas:** server, KV cache

### [Bug Fix] — PR #24305: Fix ggml-cpu rms_norm_back aliasing bug

**Merged:** 2026-06-09 | **Author:** @devYRPauli

**Summary:** Fixes a correctness bug in `ggml_compute_forward_rms_norm_back_f32` where the destination tensor could alias an input tensor. The old multi-step copy/scale/accumulate sequence overwrote the input buffer during `dx := x` and then incorrectly read it back during `+= dz`. The fix replaces this with a single fused read-before-write loop that is safe under either aliasing scenario. This affects gradient computation for RMS Norm during training/fine-tuning.

**Affected areas:** ggml-cpu, ops

### [Performance] — PR #21845: SYCL multi-column MMVQ (~45% speculative decoding speedup on Intel Arc)

**Merged:** 2026-06-05 | **Author:** @masonmilby

**Summary:** Ports the multi-column MMVQ optimization from the CUDA backend to SYCL, enabling the matrix-vector quantization kernel to compute all columns in a single dispatch instead of launching a separate kernel per column. Also relaxes `should_reorder_tensor` from `ne[1] == 1` to `ne[1] <= 8` to enable the reorder-multicol kernel path. Benchmarked on 2× Intel Arc Pro B70 with Qwen3.6-27B: speculative decoding goes from -12.3% regressive to +40.1% speedup at Q4_K_XL and +95.7% at Q8_K_XL. This is a major win for Intel Arc GPU users doing speculative decoding with MTP.

**Affected areas:** ggml-sycl, mmvq

### [Performance] — PR #24125: Fix Step35 MTP KV cache allocation for all layers

**Merged:** 2026-06-04 | **Author:** @forforever73

**Summary:** Fixes a memory waste in Step3.5 (STEP35) MTP speculative decoding where the draft context allocated KV cache for all layers even though it only runs the NextN blocks. Adds the same per-context KV layer filter that Qwen3.5 already uses: the MTP context now keeps only NextN blocks (`il >= n_main`) and the main context keeps the trunk (`il < n_main`). This reduces draft KV cache from 216 MiB to just 18 MiB (and main trunk from 1644 MiB to 0 MiB for the draft context) in a Step3.7 model test. Users of Step35/MTP models will see lower memory usage.

**Affected areas:** model, KV cache, speculative decoding
