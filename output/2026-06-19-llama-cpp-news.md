### [Bug Fix] — PR #24786: server: fix non-bound n_discard value (ctx shifting)

**Merged:** 2026-06-19 | **Author:** @ngxson

**Summary:** Fixes a high-severity server crash (unauthenticated remote DoS) when a `/completion` request with `--context-shift` enabled passes an extremely large `n_discard` value (e.g., `INT32_MAX`). The unclamped value triggers either a `std::length_error` from `size_t` arithmetic wrap-around or a `GGML_ABORT` when the full tail is removed from recurrent/hybrid memory sequences. The fix clamps `n_discard` to `n_left - 1`, ensuring at least one token is kept after `n_keep`, closing both crash paths. All server operators using `--context-shift` should upgrade — a single malicious or malformed request could crash the server.

**Affected areas:** server, context shifting, context-shift, n_discard, crash fix

### [Performance] — PR #24753: ggml-cpu: support K tails in power10 Q8/Q4 MMA matmul

**Merged:** 2026-06-19 | **Author:** @shalinib-ibm

**Summary:** Removes the requirement that the K dimension be divisible by `kc` in the POWER10 `tinyBlas_Q0_PPC` tiled matmul path for Q8_0 and Q4_0 quantization. Previously, workloads with non-multiples of `kc` fell back to the slower mnpack path; now the final K panel is processed at its actual depth through packing and kernel execution. This delivers ~60% prompt processing speed improvement with granite-3.38b-instruct (Q8_0 and Q4_0) on POWER10 ppc64le systems. This is a significant optimization for IBM Power architecture users running quantized models.

**Affected areas:** ggml-cpu, POWER10, MMA, matmul, Q8_0, Q4_0, ppc64le
