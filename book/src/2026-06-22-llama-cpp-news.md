### [CLI/Tooling] — PR #24869: common/peg: Implement AC parser for stricter grammar generation

**Merged:** 2026-06-21 | **Author:** @aldehir

**Summary:** Implements a new `ac` parser type in the common PEG grammar system that uses Aho-Corasick automaton matching to consume all characters up to and including a delimiter. This addresses a persistent issue where grammar-constrained generation could "escape" the grammar by matching partial delimiter prefixes. The new parser wraps a child parser and emits a GBNF grammar that terminates on the first complete occurrence of a delimiter (e.g., `\n</parameter>\n`), making tool-use and structured output generation more reliable. Users of grammar-constrained generation (function calling, structured output) will see fewer grammar escape failures.

**Affected areas:** PEG grammar, GBNF generation, structured output, tool use

### [Server/API] — PR #24893: Fix edit_file crash on append at end of file

**Merged:** 2026-06-22 | **Author:** @ServeurpersoCom

**Summary:** Fixes a heap-buffer-overflow crash in the `edit_file` built-in tool when the model appends content at the end of a file by sending `line_start: -1`. The off-by-one error in line normalization caused insertion beyond the buffer end. The fix restricts `-1` to append-only operations (rejecting it for replace/delete) and inserts at `end()`, correctly handling empty files. This was validated under ASan/UBSan and fixes issue #23246. Users running llama-server with the `edit_file` tool will no longer experience server crashes when models append to existing files.

**Affected areas:** server, built-in tools, edit_file
