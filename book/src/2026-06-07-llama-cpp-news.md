### [Backend] — PR #23792: TP: quantized KV cache support

**Merged:** 2026-06-01 | **Author:** @JohannesGaessler

**Summary:** Enables quantized KV cache (`--type-k`, `--type-v` flags) to work alongside tensor-parallelism mode (`--split-mode tensor`). Previously, using quantized KV cache with TP would fail due to the meta backend losing shape information during tensor flattening for KV cache rotation. The fix extends the meta backend's segment specification to track data layout through reshape operations. TP users can now reduce KV cache VRAM by adding flags like `--type-k q8_0 --type-v q8_0` to multi-GPU tensor-split setups. No changes are needed for layer-split mode.

**Affected areas:** ggml backend (meta/cuda), tensor parallelism, KV cache

### [Bug Fix] — PR #24030: Avoid PDL race conditions by disabling `__restrict__` when PDL is used

**Merged:** 2026-06-03 | **Author:** @aendk

**Summary:** Fixes a subtle race condition on NVIDIA Blackwell GPUs where `__restrict__` pointer annotations could cause data loads to be hoisted past the PDL (Persistent Dependency Latch) synchronization barrier. The fix disables `__restrict__` for device code using PDL while retaining it for all other GPU architectures. This restores the flash-attention kernels that were previously disabled on Blackwell as a workaround, yielding a ~1% performance improvement on Qwen3.6. Users with Blackwell GPUs should see improved stability and slightly better performance.

**Affected areas:** CUDA backend, Blackwell GPU, PDL synchronization, flash attention
