### [Bug Fix] — PR #25245: vulkan — fix 32-bit integer overflow in CEIL_DIV

**Merged:** 2026-07-06 | **Author:** @hokanosekai

**Summary:** Fixes a critical crash on mobile Vulkan devices caused by 32-bit integer overflow in the `CEIL_DIV` macro. Mobile GPU drivers (Mali, Adreno, CIX) report `maxComputeWorkGroupCount = UINT32_MAX`, causing the `(M + N - 1)` numerator in `CEIL_DIV` to wrap around to 0 in 32-bit arithmetic. This caused `ggml_vk_matmul` to request zero descriptor sets while still dispatching a compute shader, tripping `GGML_ASSERT(descriptor_set_idx < descriptor_sets.size())` at model load. The fix rewrites the macro with division-based math that has no overflowing intermediate, as suggested by @jeffbolznv. Tested on Mali-G68 (MediaTek Dimensity 900) — Gemma 3n E2B Q4_K_M with full Vulkan offload went from a reproducible abort at warmup to generating normally. Mobile users with Vulkan-capable GPUs from ARM (Mali), Qualcomm (Adreno), or Intel (CIX) will no longer experience immediate crashes when loading models with full GPU offload.

**Affected areas:** Vulkan backend, ggml-vulkan, CEIL_DIV macro, mobile GPU compatibility
