### [Backend] — PR #21565: metal: add CONV_2D_DW (depthwise convolution) support

**Merged:** 2026-07-09 | **Author:** @Sou-ly

**Summary:** Adds Metal GPU acceleration for `GGML_OP_CONV_2D_DW` (depthwise 2D convolution), used by MobileNet/EfficientNet-style computer vision architectures. The implementation supports both WHCN (contiguous) and CWHN (channels-first) memory layouts via byte-stride indexing, along with f32 and f16 kernel weights with f32 input/output. New tests are included for performance benchmarking across both layouts.

### [Backend] — PR #25460: cuda: align snake fusion matcher with the other backends

**Merged:** 2026-07-09 | **Author:** @ServeurpersoCom

**Summary:** Fixes two issues in the CUDA snake fusion matcher that were out of sync with the CPU, Vulkan, and Metal backends. The matcher previously required `a->type == x->type` while the kernel always reads `a` and `inv_b` as `const float*` — meaning F16/BF16 chains never fused and silently fell back to the naive path. Additionally, the matcher lacked contiguity guards to reject non-contiguous operands, which the other backends already enforced. This PR adds an F32 type check and stride-based contiguity checks, ensuring the fused path is correctly exercised for the cases it was designed to handle.

**Affected areas:** CUDA backend, snake fusion, ggml

### [Performance] — PR #24362: vulkan: disable FA mask_opt on GCN to improve performance

**Merged:** 2026-07-08 | **Author:** @0cc4m

**Summary:** Disables the flash attention `mask_opt` optimization on AMD GCN (Graphics Core Next) architecture GPUs running under the Vulkan backend. Testing on an AMD Radeon Pro VII (GCN) with RADV showed that `mask_opt` actually degrades prefill performance significantly at larger context sizes — for example, llama 8B at `pp512 @ d8192` saw a +41.1% improvement after disabling it. The effect varies by model architecture: MoE and DeepSeek models showed minor regressions at larger contexts, so the change is gated to GCN hardware only, leaving the optimization active for newer AMD architectures and NVIDIA GPUs.

**Affected areas:** Vulkan backend, flash attention, AMD GCN GPUs, mask optimization

### [Server/API] — PR #25420: server: respect min-step when splitting prompt batches

**Merged:** 2026-07-09 | **Author:** @aldehir

**Summary:** Fixes a server checkpoint logic issue where checkpoints were being triggered on every user message start regardless of the `--checkpoint-min-step` setting. The server was breaking (creating a new checkpoint) at the beginning of every user message during prompt processing, which did not respect the configured minimum step interval and negatively impacted prefill speed. This PR aligns the splitting behavior with `--checkpoint-min-step`, so checkpoints are now only created when the configured minimum step count has been reached. Fixes issues #25213 and #25320.

**Affected areas:** server, checkpoint logic, --checkpoint-min-step flag, prefill batching
