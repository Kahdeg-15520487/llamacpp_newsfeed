### [Backend/Performance] — PR #24152: SYCL --split-mode tensor

**Merged:** 2026-06-25 | **Author:** @Spruill-1

**Summary:** Adds backend-specific all-reduce (`comm_init`/`comm_free`/`comm_allreduce_tensor`) to the SYCL backend, enabling `--split-mode tensor` for multi-GPU setups on Intel Arc and other SYCL-compatible GPUs. For dual-GPU configurations, it implements a degenerate ring all-reduce with two paths: FP32 direct memcpy for small tensors (<32K elements) and BF16-compressed all-reduce for large tensors that halves PCIe bandwidth usage. This is a major multi-GPU performance unlock for Intel Arc users, enabling 70B-class models to run efficiently across dual GPUs.

**Affected areas:** SYCL backend, multi-GPU, tensor parallelism, all-reduce

### [Bug Fix] — PR #24935: llama: synchronize context before backend teardown

**Merged:** 2026-06-25 | **Author:** @krystophny

**Summary:** Fixes a crash during multi-GPU CUDA teardown where `~llama_context()` would try to free backend buffers without first draining outstanding GPU work, causing `cudaFree: unspecified launch failure` on Blackwell GPUs (RTX 5060 Ti). The fix adds a `synchronize()` call at the start of the destructor, ensuring all pending GPU operations complete before any buffer is freed. This fixes the `test-thread-safety` test on Blackwell-class GPUs and improves teardown stability for all multi-GPU CUDA users.

**Affected areas:** CUDA backend, multi-GPU, destructor ordering, Blackwell GPUs
