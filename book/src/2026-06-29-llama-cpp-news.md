## 🔍 PR Details

### [Model Support] — PR #24162: DeepSeek V4

**Merged:** 2026-06-29 | **Author:** @am17an

**Summary:** Adds full support for the DeepSeek V4 model architecture, the most significant model addition in weeks. DeepSeek V4 introduces two novel compressed attention mechanisms: CSA (Compressed Sparse Attention, compressing every 4 tokens into 1 with a window of 8 tokens) and HCA (Heavily Compressed Attention, compressing at 128-token granularity with SWA). The implementation creates "compression plans" executed on the GPU, with two new `llama_kv_cache` objects that are stream-aware and non-unified. MTP, `-sm tensor`, and the Lightning Indexer are marked as future TODOs, but the core architecture already runs on CUDA — tested on a DGX Spark (GB10) with a 284B parameter IQ2_XXS quant achieving 148 t/s prefill and 6 t/s generation. This is a landmark PR for anyone wanting to run DeepSeek's latest architecture locally.

**Affected areas:** DeepSeek V4 architecture, compressed attention (CSA/HCA), CUDA backend, KV cache management, GGUF conversion

### [Build/CI] — PR #24879: tools/ui: restore Tailwind scanning in ignored worktrees

**Merged:** 2026-06-29 | **Author:** @seryogakovalyov

**Summary:** Fixes a bug where the llama.cpp Web UI would render as unstyled DOM when built from a packaging workspace (e.g., AUR) whose root `.gitignore` ignores the UI source tree. Tailwind v4's scanner honors `.gitignore`, so it would see zero source candidates and produce a CSS bundle missing all utility rules like `.flex`, `.fixed`, and spacing utilities. The fix adds a `tools/ui/src/.gitignore` containing `!*` to re-include the source tree for Tailwind's scanner, and registers this dotfile in CMake to prevent stale asset reuse. Affects anyone building llama.cpp from non-standard worktrees or packaging environments — the Web UI will now render correctly with full Tailwind styling.

**Affected areas:** tools/ui build system, Tailwind CSS scanning, CMake, .gitignore
