### [Backend] — PR #24476: [SYCL] add dev2dev memcpy by SYCL API

**Merged:** 2026-06-17 | **Author:** @arthw

**Summary:** Fixes abnormal multi-GPU output on Intel Arc GPUs by replacing the Level Zero (L0) device-to-device memory copy API with a SYCL API implementation for dGPU+dGPU scenarios. The L0 API was found to cause crashes and corruption in certain cases (e.g., issue #23612). The new SYCL-based memcpy avoids these issues while maintaining equivalent performance (~589 t/s pp512, ~61 t/s tg128 on Llama 7B Q4_0 with dual Arc770). A new environment variable `GGML_SYCL_DEV2DEV_MEMCPY` lets users switch between SYCL (default, value 0) and L0 (value 1) APIs for debugging. For iGPU+dGPU setups, host-forward mode is still used. SYCL multi-GPU users will see stable output again without performance regressions.

**Affected areas:** SYCL backend, multi-GPU, memory copy, Level Zero

### [Backend] — PR #24719: [SYCL] rename GGML_SYCL_SUPPORT_LEVEL_ZERO

**Merged:** 2026-06-18 | **Author:** @arthw

**Summary:** Renames the SYCL Level Zero environment variables to eliminate confusing naming: `GGML_SYCL_SUPPORT_LEVEL_ZERO` → `GGML_SYCL_SUPPORT_LEVEL_ZERO_API` and `GGML_SYCL_ENABLE_LEVEL_ZERO` → `GGML_SYCL_USE_LEVEL_ZERO_API`. The original names led users to think setting these to OFF would switch SYCL from Level Zero runtime to OpenCL runtime, when in reality they only control whether Level Zero API is used for memory allocation — the underlying runtime is unaffected. Documentation is updated accordingly. This is a breaking change for existing SYCL configurations — users relying on these environment variables need to update their settings with the new names.

**Affected areas:** SYCL backend, environment variables, documentation

### [Backend] — PR #24721: [SYCL] support MUL_MAT and OUT_PROD with Q1_0

**Merged:** 2026-06-18 | **Author:** @arthw

**Summary:** Adds GPU-accelerated MUL_MAT and OUT_PROD operations for the Q1_0 quantization type on the SYCL backend, dramatically speeding up inference on Intel GPUs for Q1_0 models like Bonsai-1.7B. On an Arc B60 GPU, prompt processing (pp512) goes from ~155 to ~3042 t/s (19.6× speedup) without flash attention, and token generation from ~60 to ~148 t/s (2.47×). On integrated GPUs (Ultra 7 265K Xe), pp512 improves by 3.6×. This unlocks usable Q1_0 models on Intel hardware — previously these operations fell back to the CPU for Q1_0, bottlenecking performance.

**Affected areas:** SYCL backend, Q1_0 quantization, MUL_MAT, OUT_PROD
