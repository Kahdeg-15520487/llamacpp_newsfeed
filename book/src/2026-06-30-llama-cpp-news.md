### [Backend] — PR #25143: ggml-webgpu: add support for NVFP4

**Merged:** 2026-06-30 | **Author:** @yomaytk

**Summary:** Adds NVFP4 (NVIDIA FP4) format support to the ggml-webgpu backend, enabling WebGPU to run NVFP4-quantized GGUF models natively on the GPU instead of falling back to the CPU. Performance benchmarks on Gemma 4 31B NVFP4 show dramatic improvements — on M2, prefill went from 13.61 t/s to 44.06 t/s and token generation from 0.90 t/s to 2.51 t/s; on Tesla V100, prefill jumped from 1.39 t/s to 30.38 t/s and generation from 1.04 t/s to 3.35 t/s. The primary motivation is ecosystem coverage: as NVFP4 GGUFs become increasingly available on HuggingFace (e.g., Gemma 4 family), this ensures WebGPU users can run them without CPU fallback. Note that NVFP4 offers no performance advantage over k-quants on WebGPU; this is purely a compatibility/coverage improvement.

**Affected areas:** ggml-webgpu backend, NVFP4 quantization format, WebGPU shader kernels
