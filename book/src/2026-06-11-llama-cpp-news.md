### [Bug Fix] — PR #24411: server : skip checkpoints beyond pos_next

**Merged:** 2026-06-11 | **Author:** @aldehir

**Summary:** Fixes a cache corruption bug in the server's checkpoint restore logic when using sliding-window attention (SWA) models like Gemma 4. When checkpointing during SWA inference, positions beyond `pos_next` were being unnecessarily checkpointed, leading to corrupted cache reuse on subsequent identical prompts. The fix skips checkpointing tokens past the current write position, ensuring that cache-restored responses match the original output. This resolves "forgetfulness" issues with Gemma 4 models when reusing cached sessions, and can be reproduced with any SWA model sending a repeated initial message after multiple turns.

**Affected areas:** server, checkpointing, SWA

### [Model Support] — PR #24428: vocab : adopt leading TemplateProcessing special token as BOS

**Merged:** 2026-06-11 | **Author:** @o7si

**Summary:** Fixes BERT-class models (like `bert-base-uncased`) missing their `[CLS]` prefix and `[SEP]` suffix tokens during tokenization. The root cause was that the leading special token in the `TemplateProcessing` template wasn't recognized as a BOS token when it lacked an explicit name in `tokenizer_config.json`. This PR updates the vocabulary conversion logic to adopt the leading TemplateProcessing special token as BOS, bringing llama.cpp's tokenization output in line with the HuggingFace `transformers` library. Users of BERT-family models will now see correct `[CLS]`/`[SEP]` wrapping (e.g., `hello world` → `[101, 7592, 2088, 102]` instead of just `[7592, 2088]`).

**Affected areas:** vocab, tokenizer, BERT, conversion

### [Bug Fix] — PR #24371: vocab : refactor normalizer flags into options struct, add strip_accents

**Merged:** 2026-06-11 | **Author:** @o7si

**Summary:** Fixes WordPiece Model (WPM) tokenization for case-sensitive models where `strip_accents` is set to `false`. Previously, NFD normalization was applied unconditionally during WPM tokenization, causing accented characters (like `ö`, `ü`, `é`) to be decomposed and tokenized incorrectly — producing different token IDs than HuggingFace `transformers`. This PR refactors the normalizer flags into an options struct and only applies NFD decomposition when `strip_accents` is enabled. Models such as `German_Semantic_V3` now produce correct token IDs for accented words (e.g., `schön` → `[102, 2602, 103]` instead of `[102, 778, 103]`), matching `transformers` output exactly.

**Affected areas:** vocab, normalizer, WPM, tokenization
