### [Backend] — PR #23961: ggml-cpu: add AVX2 and AVX optimization for nvfp4 dot product

**Merged:** 2026-07-01 | **Author:** @ragz4125

**Summary:** Adds CPU-optimized implementations of `ggml_vec_dot_nvfp4_q8_0` for AVX2 and AVX instruction sets, delivering a dramatic **10.7× speedup** over the baseline — from 2.85 t/s to 30.48 t/s on an Intel Core i9-7900X for a Qwen3-4B NVFP4 model at pp512. The implementation leverages the existing `GGML_CPU_UE4M3_TO_FP32` lookup table for UE4M3 scale decoding, which may also benefit other code paths using this shared infrastructure. This is a significant milestone for NVFP4 on CPU: previously NVFP4 models were severely bottlenecked on CPU-only systems, and this optimization makes them usable without a GPU. Users running NVFP4-quantized models (e.g., Gemma 4, Qwen3 NVFP4 variants) on CPU or hybrid CPU/GPU setups will benefit immediately — no new flags or configuration changes needed.

**Affected areas:** ggml-cpu backend, AVX2/AVX SIMD kernels, NVFP4 quantization format, dot product computation
