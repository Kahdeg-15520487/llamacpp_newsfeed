### [Server/API] — PR #25322: server: eagerly create prompts log directory at startup

**Merged:** 2026-07-08 | **Author:** @rankaiyx

**Summary:** The `--log-prompts-dir` server flag now automatically creates the target directory at startup using `std::filesystem::create_directories`, instead of requiring the directory to already exist before the first request. Previously, if the directory was missing, each request would trigger an `std::ofstream` open failure and an `SRV_ERR` log entry, effectively breaking the debugging feature. This is especially useful for temporary filesystems like `/tmp/foo` where manually creating the directory before each server restart is impractical.

**Affected areas:** server, --log-prompts-dir flag, prompt logging

### [Performance] — PR #25278: llama-batch: enable parallel sequences for partial rollback

**Merged:** 2026-07-08 | **Author:** @am17an

**Summary:** Enables parallel decoding for recurrent models (like Mamba-based architectures) that use partial rollback for state management. The fix ensures that the last `n_keep_tail` tokens always belong to the same ubatch in the `split_equal` function, which previously could split the tail tokens across different micro-batches and cause incorrect state rollback. Recurrent model users who rely on parallel sequence decoding will see correct behavior instead of potential state corruption, and the change is purely internal to the batch splitting logic — no new flags or user-facing changes are introduced.

**Affected areas:** llama-batch, split_equal, recurrent models, parallel decoding, partial rollback
