### [Bug Fix] — PR #25247: ggml — fix broken CPU concat implementation for quantized types

**Merged:** 2026-07-04 | **Author:** @fairydreaming

**Summary:** Fixes a critical correctness bug in the CPU concat operation (`ggml_compute_forward_concat_any()`) that caused out-of-bound memory accesses when concatenating quantized tensors (e.g., Q8_0). The existing implementation iterated over individual tensor elements using `type_size` as stride, but for quantized types, elements are packed into blocks — the inner loop's `memcpy` calls would read/write past allocated buffers. The fix iterates over whole quantization blocks instead of individual elements, assuming contiguous tensors. This is a prerequisite for DeepSeek V4 quantized KV cache support (PR #25202, upcoming). All users relying on CPU-path concat of quantized types (e.g., speculative decoding KV cache injection) will see correct behavior instead of silent memory corruption or crashes. Affects ggml-cpu/ops.cpp.

**Affected areas:** ggml CPU backend, quantized tensor concatenation, ops.cpp

### [Bug Fix] — PR #25215: llama — skip K/V rotation input when its buffer is unallocated

**Merged:** 2026-07-04 | **Author:** @liminfei-amd

**Summary:** Fixes a crash (`GGML_ASSERT(buffer) failed`) in the K/V rotation input path that occurred during DFlash speculative decoding's KV-injection pass. When a graph only stores K/V without attending (as DFlash does), certain rotation tensors have a non-null pointer but an unallocated (`NULL`) buffer. The `set_input_k_rot`/`set_input_v_rot` functions attempted to check the buffer's host status, hitting the GGML_ASSERT. The adjacent `kq_mask` inputs already guarded for this case with a `->buffer` null check — this PR adds the same guard to the four `k_rot`/`v_rot` inputs. Reproduced and fixed on RDNA4 (gfx1201, Vulkan) with DFlash draft models using head dims multiples of 64. Users running DFlash speculative decoding with `--cache-type-k q8_0` and rotation-enabled attention will no longer crash on the first draft decode.

**Affected areas:** llama KV cache, DFlash speculative decoding, K/V rotation input, Vulkan backend
